# Python Project Auditor

Performs a two-phase audit:

1. **Phase 1** — Inspect each subfolder under `src/` and write a `README.md` inside it.
2. **Phase 2** — Consolidate all findings into a single `PROJECT_AUDIT_REPORT.md` at the root.

---

## Phase 1 — Per-Folder README Generation

### Step 1: Discover folders

```bash
find src/ -mindepth 1 -maxdepth 1 -type d | sort
```

If no `src/` directory exists, ask the user which root directory to scan.

### Step 2: For each folder, inspect the code

Collect the following before writing the README:

```bash
# Directory tree
find <folder> -type f | sort

# All Python source files
find <folder> -name "*.py" | xargs cat 2>/dev/null

# Dependencies
cat <folder>/requirements*.txt <folder>/pyproject.toml <folder>/setup.py 2>/dev/null

# Existing README (to extend, not overwrite)
cat <folder>/README.md 2>/dev/null
```

### Step 3: Write README.md inside the folder

Base every section **strictly on the actual code read above**. Do not assume or hallucinate.

If the folder is not a Python project, state it explicitly at the top.

If a README already exists, improve and extend it — do not blindly overwrite.

## Phase 2 — Consolidated PROJECT_AUDIT_REPORT.md

After generating all per-folder READMEs, create `PROJECT_AUDIT_REPORT.md` at the project root.

```bash
# Read all generated READMEs as input
find src/ -name "README.md" | xargs cat
```

Base the report **only** on the folder READMEs generated in Phase 1. Do not re-read source code.

## Execution Rules

1. **Read before writing.** Always inspect actual files before drafting any section.
2. **Cite specifics.** Reference file names, function names, and line numbers when reporting issues.
3. **No assumptions.** If the code cannot be read, say so explicitly rather than guessing.
4. **Non-Python folders.** State at the top of the README that the folder is not a Python project, then briefly describe what it contains.
5. **Existing READMEs.** Merge and extend; preserve any useful content already there.
6. **Order of operations.** Always complete all per-folder READMEs before writing `PROJECT_AUDIT_REPORT.md`.
7. **Professional tone.** Write as a senior software architect: direct, specific, and actionable.
